01 Design Philosophy

This design system embodies the thesis: legitimate knowledge emerges from unexpected, democratized sources. The zine aesthetic connects TikTok's creator-educators to a longer history of grassroots publishing‚Äîzines, pamphlets, samizdat‚Äîwhere barrier-free entry and DIY production challenged institutional gatekeeping.

Core Tension: Intentional imperfection that still communicates clearly.

Guiding Principles

  1. Democratic over polished ‚Äî The aesthetic should feel accessible, not exclusive
  2. Energetic over static ‚Äî Rotation, collage, and visual "noise" create movement
  3. Mixed over uniform ‚Äî Typography and elements should feel assembled, not templated
  4. Readable over decorative ‚Äî Accessibility always wins; imperfection shouldn't impede comprehension
  5. Meaningful over random ‚Äî Every "messy" element should serve communication
Rise/Mighty Note: Rise has clean, minimal defaults. Use Mighty's CSS injection and custom blocks strategically‚Äîdon't fight every default, but disrupt key moments (headers, callouts, section transitions) to establish the zine personality.

05 Rotation & Imperfection

The "messy" zine feeling comes from controlled imperfection. Use rotation and offset deliberately.

Rotation Tokens

  • --rotate-slight: 1.5deg ‚Äî Subtle, almost subconscious tilt. Use for: body elements that need slight energy
  • --rotate-medium: 3deg ‚Äî Noticeable but not distracting. Use for: callout boxes, stickers, labels
  • --rotate-strong: 6deg ‚Äî Clear intention. Use for: hero elements, feature callouts, phone mockups

Rules for Rotation

‚úì Do
  • Rotate decorative elements (stickers, tape, stamps)
  • Tilt callout boxes and pull quotes
  • Offset elements from strict grid alignment
  • Vary rotation direction (mix positive/negative)
  • Use rotation on hover states for interactivity
‚úó Don't
  • Rotate body text blocks
  • Rotate navigation or critical UI
  • Use the same rotation angle for everything
  • Rotate to the point of illegibility
  • Overdo it‚Äîchaos should be orchestrated
Technical Note: In CSS, use transform: rotate(). Combine with box-shadow for the cut-out effect. In Figma, rotate frames and use drop shadows to simulate layering.

06 Content Guidance

Voice & Tone

The People's Professors voice is:

  • Confident but not academic ‚Äî Make claims, but accessibly
  • Enthusiastic but not hype-y ‚Äî Genuine excitement, not marketing speak
  • Direct ‚Äî Short sentences. Clear points. No hedging.
  • Occasionally irreverent ‚Äî Emoji, asides, personality welcome

Writing Style

‚úì Do
  • Use active voice
  • Front-load key information
  • Break up long passages with visuals
  • Use annotations/marginalia for asides
  • Include the occasional emoji for tone üí°
  • Reference the platform naturally (TikTok, algorithm, FYP)
‚úó Don't
  • Use jargon without defining it
  • Write dense academic paragraphs
  • Be condescending about the subject matter
  • Over-explain or pad content
  • Use corporate/marketing language

Content Structure (for Rise)

Recommended lesson flow:

  1. Hook ‚Äî Bold statement, question, or provocative framing (use Marker font)
  2. Context ‚Äî Brief setup, 2‚Äì3 sentences max
  3. Core Content ‚Äî Chunked into digestible blocks
  4. Example/Evidence ‚Äî Embedded video or case study
  5. Application ‚Äî How does this connect to their work?
  6. Takeaway ‚Äî One clear thing to remember (use callout component)

07 Accessibility

The zine aesthetic can coexist with accessibility. Here's how:

Non-Negotiables

  • Color contrast: All text must meet WCAG AA (4.5:1 for body, 3:1 for large text)
  • Don't convey info by color alone: Pair with icons, labels, or patterns
  • Keyboard navigation: All interactive elements must be focusable
  • Alt text: Decorative elements can be empty alt, but meaningful images need descriptions
  • Reduced motion: Respect prefers-reduced-motion for any animations

Rotation & Screen Readers

CSS rotation is purely visual‚Äîscreen readers read content in DOM order regardless of rotation. The only concern is if rotation causes text clipping or overflow issues.

Rise Note: Rise has solid baseline accessibility. When adding custom HTML/CSS, test with keyboard navigation and VoiceOver/NVDA to ensure you haven't broken anything.
